# 07 - Healthy Technology
## Bullet Point Version

## One-line charter

Technology should give attention back.

## What that means

- Use screens as doorways, not destinations.
- Build prompts that send people back to Frankston.
- Help people notice, walk, draw, write, make, discuss, print, teach, and share.
- Keep digital sessions short.
- Give every experience a clear ending.
- Make tools understandable to parents, teachers, librarians, and council officers.
- Measure only what is necessary.
- Protect children by default.

## Hard no

- No infinite scroll.
- No dark patterns.
- No gambling mechanics.
- No loot boxes.
- No streak pressure.
- No compulsory accounts for public participation.
- No personal data collection from children.
- No open child profiles.
- No unmoderated child submissions.
- No hidden tracking.
- No public leaderboards for children.
- No app install requirement for basic access.
- No cultural content without permission and appropriate guidance.

## Required design behaviours

- No-login by default.
- Short sessions.
- Clear exits.
- Offline prompts.
- Family mode.
- Classroom mode.
- Print mode.
- Plain English.
- Accessible contrast.
- Mobile readable.
- Keyboard navigable where interactive.
- Alt text for meaningful images.
- Captions/transcripts for audio or video.
- Anonymous aggregate metrics only where needed.

## Create, then leave the screen

Every page should move toward an offline action:

- draw this
- walk here
- notice this
- talk about this
- print this
- take this to class
- make this with family
- bring this to the library
- add this to a workshop

## Design rules

- One tool, one purpose.
- One prompt, one next action.
- No feature without a public value.
- No data without a reason.
- No metric without a decision it supports.
- No child-facing account unless a reviewed adult-managed pathway requires it.
- No engagement loop designed to keep children on screen.
- No decorative technology that does not help the place.

## Risk controls

- Replace child submissions with teacher-led or adult-mediated options.
- Replace personal accounts with no-login access.
- Replace live commenting with curated workshop outputs.
- Replace leaderboards with group reflection.
- Replace endless galleries with finite poster sets.
- Replace push notifications with printed reminders or adult-managed updates.
- Replace always-online tools with printable worksheets.
- Replace rich tracking with anonymous aggregate counts.

## Pilot checklist

- Purpose is clear in one sentence.
- User group is defined.
- No-login version exists.
- Child-data position is explicit.
- Offline action is included.
- Clear ending is visible.
- Print version exists where useful.
- Family/classroom mode is available where relevant.
- Accessibility basics checked.
- Permission needs identified.
- Metrics are minimal.
- Stop conditions are written down.

## What Council can support

- No-login public art QR pages.
- Printable lesson and poster packs.
- Library workshop cards.
- School-friendly classroom prompts.
- Family walking prompts.
- Anonymous pilot reporting.
- Adult-mediated participation pathways.

## What B$S must prove

- It can build useful tools without attention traps.
- It can protect children by design.
- It can keep data collection minimal.
- It can provide offline alternatives.
- It can write plain public instructions.
- It can support Council, schools, libraries, and families.
- It can keep Frankston as the hero.

## Success

- People make something offline.
- Teachers can use it.
- Libraries can host it.
- Families can understand it.
- Council can explain it.
- Public art becomes more useful.
- Frankston becomes more creative.

## Failure

- The screen becomes the main event.
- Children are asked for personal data.
- Accounts become the gate.
- Engagement replaces creativity.
- Metrics replace meaning.
- The project becomes brand theatre.

## Ruthless rule

- If it steals attention, cut it.
- If it needs child data, redesign it.
- If it encourages compulsive use, reject it.
- If it cannot be explained to a parent in one sentence, simplify it.
