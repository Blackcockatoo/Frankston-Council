# 08 - Pilot Projects
## Visual Layout Suggestions

## Purpose

Make the pilots skimmable for councillors and practical for officers. The page should not feel like a brainstorm. It should feel like a controlled decision set.

## Recommended structure

### Page 1 - Pilot menu

- Five pilot cards.
- Each card: title, one-line purpose, cost band, risk level, best first audience.
- Highlight one recommended first candidate: Living Mural / QR Public Art Layer.

### Page 2 - 30/60/90 timeline

- Horizontal timeline.
- Three columns: 30 days, 60 days, 90 days.
- Each pilot gets one row.
- Use restrained colour, not decorative icons.

### Page 3 - Pilot detail template

For each pilot:

- Purpose
- What it looks like
- Who it serves
- Council provides
- Blue Snake Studio provides
- Risks
- Safeguards
- Success metrics
- Next decision

### Page 4 - Risk gate

- A simple "Can this proceed?" checklist:
  - Permission clear?
  - Child safety clear?
  - Privacy clear?
  - Cultural protocol clear?
  - Owner clear?
  - Maintenance clear?

### Page 5 - Recommended Stage One path

- Large line: "Approve exploration of one small scoped pilot."
- Three steps:
  1. Choose one candidate.
  2. Scope permissions and risk.
  3. Build/test/decide.

## Visual tone

- Civic, calm, readable.
- Use the Frankston palette from the constitution.
- Use diagrams before paragraphs.
- Do not make the pilots look like product launches.
- Do not use mystical Blue Snake Studio branding.
- Use Blue Snake Studio only as a small maker/delivery credit.

## Do not do

- Do not show First Nations icons/symbols.
- Do not use Japanese decorative motifs for Susono content.
- Do not make QR codes decorative.
- Do not imply council endorsement.
- Do not make all five pilots look equally recommended.
