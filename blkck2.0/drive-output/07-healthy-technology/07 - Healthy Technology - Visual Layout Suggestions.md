# 07 - Healthy Technology
## Visual Layout Suggestions

## Design intent

The Healthy Technology Charter should feel formal enough for Council and clear enough for teachers, librarians, parents, artists, and community workers.

Visual tone:

- civic
- calm
- direct
- practical
- slightly street-wise through layout rhythm, not gimmick graphics

Avoid making the charter look like a tech brand manifesto. Frankston remains the hero.

## Recommended format

Create three visual outputs:

1. A formal 6-8 page charter PDF.
2. A 1-page council/school summary.
3. A poster-style public version for workshops and libraries.

## Suggested page sequence for formal charter

### Page 1 - Cover

Title:

Healthy Technology Charter

Subtitle:

Technology should give attention back.

Footer:

Frankston 2035 - Art That Lives

Small maker stamp:

Prepared by Blue Snake Studio

### Page 2 - Definition

Use a large central statement:

Healthy technology helps people notice, make, learn, connect, and participate - then lets them leave the screen.

Include a simple four-step flow:

Open -> Prompt -> Make Offline -> Finish

### Page 3 - Non-negotiables

Use a clean two-column checklist:

- No infinite scroll
- No dark patterns
- No gambling mechanics
- No login by default
- No child personal data collection
- Short sessions
- Clear endings
- Offline creative prompts
- Family/classroom modes
- Accessibility basics
- Anonymous aggregate metrics only where needed

### Page 4 - Who this protects

Use five labelled panels:

- children
- families
- schools
- libraries
- Council

Each panel should have one practical protection statement.

### Page 5 - Design rules

Use rule cards or a compact table:

- one purpose per tool
- create, then leave the screen
- build the exit
- no data without a reason
- print version where useful
- explain it to a parent in one sentence

### Page 6 - Pilot approval checklist

Use a tick-box layout so the page can become a working review form.

### Page 7 - Success and failure

Split page:

Success looks like / Failure looks like

Keep this blunt and council-readable.

### Page 8 - Next actions

Show three immediate actions:

1. Use charter to review first pilot.
2. Draft public privacy note.
3. Create no-login prototype with offline prompt.

## Colour and type

Suggested palette:

- deep blue/black base for headers
- bay light for calm background areas
- cockatoo yellow for restrained highlights and tick marks
- clean white space for readability

Avoid:

- heavy gradients
- mystical branding
- overly dark pages
- neon overload
- tech startup gloss

Typography:

- bold readable headings
- simple body copy
- generous line spacing
- strong hierarchy
- no gimmick fonts

## Motifs to use lightly

- walking trail line
- QR portal outline
- worksheet tick marks
- poster wall grid
- bay horizon line
- simple black cockatoo reference only if already approved as a visual motif

Do not invent cultural symbols or borrow First Nations visual language.

## Layout motifs

### The attention-back loop

Use a simple loop:

Place -> Prompt -> Make -> Share/Reflect -> Place

The loop should return to place, not to screen.

### The exit marker

Every example screen should include an obvious ending marker:

"Close this and draw."

"Walk to the next site."

"Print this for class."

### The rejected-pattern strip

Create a narrow strip showing what is not allowed:

infinite scroll / dark patterns / gambling mechanics / child data / default login

Use restraint. This should read as a safety standard, not as shock design.

## Accessibility notes

Visual design should include:

- high contrast text
- minimum practical body size for print
- clear headings
- no text over busy images
- alt text plan for web images
- print-friendly black and white version
- PDF export tested for readability

## Public poster treatment

Poster headline:

Create, then leave the screen.

Body:

Healthy technology helps you notice Frankston, make something, and return to the real world with more attention.

Footer:

Frankston 2035 - Art That Lives

Small maker stamp:

B$S / delivery engine

## Web treatment

On the website, treat the charter as:

- a short public explainer
- a download link
- a rules block
- a pilot checklist
- a council/school assurance point

Do not hide the practical rules below a decorative hero. The rules are the value.
