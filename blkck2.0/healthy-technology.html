Healthy Technology Charter

Technology should give attention back.

Frankston 2035 uses digital tools only when they help children, families, schools, libraries, artists, and public places become more creative offline.

Core principle

Create, then leave the screen.

The screen is a doorway. A good QR or web layer gives a short prompt, supports an offline action, offers optional reflection, then finishes clearly.

If a feature steals attention, cut it. If it needs child data, redesign it. If it encourages compulsive use, reject it.

Design rules

Non-negotiables for child-facing pilots.

No infinite scroll

Every session should have a clear end.

No dark patterns

No tricks, countdown pressure, fake scarcity, or manipulative prompts.

No gambling mechanics

No streaks, loot boxes, random rewards, or compulsive loops.

No login by default

Use anonymous access unless a reviewed adult-led mode requires otherwise.

No child personal data

Do not collect unnecessary names, contacts, accounts, photos, or identifiers.

Printable fallback

Every QR path needs an offline or paper alternative.

Screen as doorway

QR / Link -> Short Prompt -> Offline Action -> Optional Reflection -> Finish.

  1. Load quickly on a phone.
  2. Explain the activity in one sentence.
  3. Ask the user to look, move, sketch, talk, write, listen, or make.
  4. Let families or classrooms reflect without public posting.
  5. End with a clear exit prompt.

Proof checklist

No-login test, no-child-data test, printable fallback, mobile readability, accessibility pass, visible privacy note, clear exit prompt, adult-led group mode.

Council, schools, libraries, and parents need proof that the technology is useful and bounded. The charter is not enough; the demo has to behave this way.
