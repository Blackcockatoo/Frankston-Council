# Foundation — Visual Layout Suggestions

How this constitution should *look* when it (or anything built from it) becomes a real page.

---

**For this document itself, if produced as a PDF:**
- Cover page: project title in serif display, large, centred on `paper`, with the single memetic line beneath it in smaller sans. No imagery — this is the constitution, it should feel like a clean civic document, closer to a charter than a brochure.
- Numbered sections (1–17) rendered as a left-margin running index down the side of each page — readers should always know where they are in the document.
- Section 13 (Colour & Typography) should visually demonstrate itself: actual coloured swatches with hex codes, actual rendered type samples, not just a description.
- Section 16 (Folder Structure) rendered as a simple nested-box diagram, not a code block, when it appears in design-facing (rather than developer-facing) versions.

**For any document built FROM this constitution (the general pattern):**
- Open with the single message in serif display, large, with generous white space above and below — this is the "hero is a thesis" rule.
- Follow immediately with a one-line "why this matters to Frankston" — small caps or coloured eyebrow text above the next block.
- Use a simple horizontal timeline/diagram graphic (bay-mid circles, bay-pale connecting line) wherever a roadmap or staged process is described — never describe staging in prose alone.
- Pull-quotes (from Section 12's guiding quotes bank) set in serif display, dusk-coral text or a dusk-coral rule beside them, isolated in their own white-space block — used as page-break moments in longer documents, not squeezed beside other content.
- Tables (like the Audience Profiles or Success Metrics tables in this document) rendered with bay-pale header rows, generous cell padding, hairline `bay-mid` borders — never heavy black grid lines.
- Footer credit line on every page: small, sans, `bay-mid` text — "A proposal by Blue Snake Studio" — never larger or more prominent than any Council or Traditional Owner mark present on the page.

**Layout don'ts:**
- No gradient backgrounds.
- No stock photography of generic people in suits shaking hands.
- No more than one dusk-coral element per page.
- No dense walls of body text under a heading — if a section runs long, break it with a diagram, a pull-quote, or a table before continuing.
