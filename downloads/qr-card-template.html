QR Card Template

Placeholder cards — QR codes pending stable URLs.

These are layout templates for print-and-apply QR trail cards. The QR code boxes are intentionally blank: actual QR codes must only be generated once the destination URLs are confirmed stable and reviewed for safety.

Do not generate or print live QR codes until: destination URLs are confirmed, the privacy layer is reviewed, and the pilot scope is approved. A QR code linking to a broken or unapproved destination creates public confusion and risks the pilot.

What's needed before printing live cards

Three gates before QR codes go on walls.

  1. Stable URL. The destination page must be on a confirmed, permanent URL — not a development or preview URL. Vercel preview URLs change on every deploy.
  2. Content review. The landing page the QR code points to must be reviewed for: no student personal data, clear ending, no login required, accessible on mobile without an app.
  3. Pilot scope confirmation. QR codes placed on physical public infrastructure (walls, trail signs, library displays) need council or venue permission for the location and duration.

Until these three gates are cleared, use the placeholder layout below to plan card dimensions, text, and position — but do not print for public use.

Safe QR layer rules

Every QR code must pass these checks.

  • The destination URL is permanent and will not change when code is republished.
  • The destination page loads on mobile without an app install.
  • The destination page requires no login by default.
  • The destination page collects no personal data from visitors.
  • The destination page has a clear ending — a task done, a thing made, a place noticed.
  • The destination page is accessible: readable text size, alt text on images, works without JavaScript for core content.
  • An adult (teacher, council officer, or studio) can remove or redirect the QR code quickly if needed.

See the full rules at Healthy Technology Charter.

Next step for QR cards

Confirm pilot scope and URL before generating any real codes.

When the pilot location and scope are confirmed:

  1. Lock the destination URL in Vercel (check cleanUrls config is stable).
  2. Generate QR codes using a free tool (e.g., qr-code-generator.com) — do not use tools that embed tracking pixels or require an account for the free tier.
  3. Test each code on at least two different phones before printing.
  4. Print at minimum 3×3 cm for reliable scanning; larger if the card will be at arm's length or above.
  5. Add a short plain-text URL under each code so it can be typed if the scan fails.
